Welcome to the Project Plan

Plan Goal

Our project plan is to have a new minigame created every two weeks and have debugging done on said game within the following third week. Chat messaging for the game will be incorporated during the debugging phase. So, by every third week, there will be a functioning game with a chat function added. While the library will not be heavily populated with games by the end of the semester, the games that will be present will be fully operational. For the sake of concept, we are going for quality over quantity. In time, this should lead to a well-designed website.

If Time Permits

If time permits, we would like to add a search bar so users will be able to find a specific game in the library. We would also like to implement a way for users to message images. Both are minor things, but would greatly enhance the users' experience while using the webapp. We are not sure if these are all the features that we would like to add, but we know this would be a good general starting point for improvements.

With Time Constraints

If the time is lost, the features that we may have to let go would be quantity of minigames. Our goal is to have at least five minigames a player can switch through while using our web app. If there isn't enough time, we'll only shoot for four, or three, we really hope to have a minimum of three games.
